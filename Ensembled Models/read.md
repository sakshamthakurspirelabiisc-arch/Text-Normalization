## Ensemble_Models

This directory contains the training and inference code for the ensemble-based text normalization system.

The ensemble combines predictions from:

* **Rule-Based Model (RB)**
* **mT5 Small**
* **Hints_IB (IndicBERT with Hints)**

using two complementary routing approaches:

### 1. Sentence-Level Router

The sentence-level router selects a single model whose output is predicted to be the most accurate for the entire input sentence.

The task is formulated as a multi-class classification problem, where the classes correspond to the candidate normalization models (RB, mT5 Small, and Hints_IB).

For each training example, the Character Error Rate (CER) between the output of each model and the ground-truth normalized sentence is computed. The model(s) with the lowest CER are considered the correct class labels.

Since multiple models can achieve the same lowest CER for a sentence, training is performed using an average classification loss over all valid classes. Specifically, the loss is computed assuming each lowest-CER model is the correct class, and the resulting losses are averaged. This allows the router to learn from examples where multiple models perform equally well.

### 2. Span-Level Router

The span-level router operates at the level of individual unnormalized spans rather than the entire sentence.

For each detected unnormalized span:

1. Candidate normalizations are obtained from all constituent models.
2. The CER between each candidate span normalization and the corresponding ground-truth span is computed.
3. The model(s) producing the lowest CER for that span are treated as the correct class labels.

Similar to the sentence-level router, span-level routing is formulated as a classification task with support for multiple valid classes when multiple models achieve the same minimum CER.

During inference, the span-level router selects the most suitable model prediction for each span independently. The selected span normalizations are then combined to construct the final normalized sentence.



---

## Training Data Requirements

Training the sentence-level and span-level routers requires:

* The unnormalized input sentence.
* The ground-truth normalized sentence.
* Predictions generated by each constituent model (RB, mT5 Small, and Hints_IB) for the same input sentence.

In other words, for every training example, the outputs of all participating normalization models must be available alongside the reference normalized sentence.

### Example

```python
{
    "unnorm": "मेरी उम्र 25 साल है",

    "rb_op": "मेरी उम्र पच्चीस साल है",

    "mt5_op": "मेरी उम्र पच्चीस साल है",

    "hints_ib_op": "मेरी उम्र पच्चीस साल है",

    "normalized_op": "मेरी उम्र पच्चीस साल है"
}
```

The routers automatically determine the best-performing model(s) by comparing each model output against the ground-truth normalized sentence using Character Error Rate (CER).

## Folder Contents

This directory includes:

* Training code for the sentence-level router.
* Training code for the span-level router.
* Inference code for the complete ensemble system for both type of dynamic routers



The final ensemble system combines the strengths of Rule-Based normalization, mT5 Small, and Hints_IB to achieve improved text normalization performance over any individual model.
